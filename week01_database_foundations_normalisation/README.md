# Week 1 — Database Foundations and Normalisation

This week focuses on planning and designing a relational database before any SQL
queries are written. The emphasis is on understanding the business scenario,
defining entities and relationships, and applying normalisation principles to
produce a clean and reliable schema.

The main goal was to understand how good database design prevents redundancy,
inconsistency, and logic errors later on.

All work for this week was completed as a conceptual database design exercise.
---

## What I Worked On

This week covered:

- defining a business scenario and data requirements
- identifying entities and attributes
- assigning primary and foreign keys
- modelling one-to-many and many-to-many relationships
- understanding and applying normalisation (1NF, 2NF, and 3NF)
- designing a relational structure that supports future querying

The main goal was to ensure that each table represents a single concept and that
each piece of data is stored in the correct place.

## Notes

This week emphasised that good SQL starts long before writing queries.
A well-designed schema reduces redundancy, prevents data anomalies, and makes
later querying simpler and more reliable.

Understanding normalisation at this stage felt similar to experimental design:
errors made early are much harder to correct later.
